# 舆情分析

对采集到的社交媒体内容进行结构化分析，输出可操作的舆情洞察报告。

## 工作流程

### Step 1：确认分析目标

开始前，向用户确认以下信息（未提供则询问）：

- **监测对象**：品牌名 / 产品名 / 竞品名称
- **数据来源**：已有数据文件路径，或需要先采集（→ 调用 `xiaohongshu-justoneapi` skill）
- **时间范围**：数据的时间区间
- **分析侧重**：整体口碑 / 危机排查 / 竞品对比 / 营销效果评估

### Step 2：数据预处理

数据量超过 200 条时，先运行预处理脚本将大文件压缩为分析摘要：

```bash
uv run skills/sentiment-analysis/scripts/prepare.py <input_csv> [--top N] [--output summary.json]
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `input_csv` | 输入文件（CSV/JSON），支持 xiaohongshu-justoneapi 的输出格式 | 必填 |
| `--top N` | 每个情感类别保留 top N 条（按互动量排序） | 30 |
| `--output` | 输出摘要文件路径 | `sentiment_summary.json` |

脚本输出的 `sentiment_summary.json` 即为下一步分析的输入。

### Step 3：五维度分析

读取摘要数据后，从以下五个维度展开分析：

#### 1. 情感分布
- 将每条内容判定为：**正面** / **中性** / **负面**
- 计算三者占比，识别整体情绪基调
- 摘取各类别最典型的 2-3 条原文作为佐证

#### 2. 热点话题聚类
- 识别高频出现的主题（功效、成分、使用体验、价格、包装等）
- 每个主题列出代表性内容
- 标出正向话题 vs 负向话题

#### 3. 互动信号挖掘
- 提取点赞/评论/收藏量最高的内容
- 分析高互动内容的共同特征（共鸣点是什么）
- 识别真实用户需求与痛点

#### 4. KOL 识别
- 找出互动量显著高于均值的账号
- 标注其内容倾向（正面/负面/中立）
- 评估对品牌的潜在影响力

#### 5. 危机预警
- 识别情绪激烈的负面内容（投诉、避雷、质量问题等）
- 检测近期负面趋势是否有加速迹象
- 标出需要立即关注的高风险内容

### Step 4：生成分析报告

读取 `templates/report.md` 作为输出结构，填充分析结果。报告要求：

- **每个结论必须有数据支撑**（引用原文、给出占比或互动量）
- **行动建议要具体可执行**，避免"加强监控"这类空话
- **执行摘要放在最前**，让不看细节的人也能立刻获取关键判断

## 数据格式兼容性

本 Skill 兼容以下数据格式：

| 来源 | 字段映射 |
|------|---------|
| `xiaohongshu-justoneapi` (CSV) | `title`、`desc`、`likes`、`comments_count`、`collected_count`、`create_time_fmt` |
| 通用 JSON | 需包含 `text`（正文）、`engagement`（互动数）、`created_at`（时间） |

字段不完整时，跳过缺失维度的分析并在报告中注明。
